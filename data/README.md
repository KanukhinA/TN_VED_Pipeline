# Данные для сборки

## Полный классификатор ТН ВЭД в интерфейсе

Листбокс на шаге «Классы» строится из **`frontend/src/catalog/tnVedChildren.generated.ts`**. Пока вы не пересоберёте его из **полного** Excel, в репозитории лежит только небольшой демо-фрагмент (одна глава).

Положите сюда **полный** файл **`ТН ВЭД.xlsx`** (все нужные вам коды и уровни, как в официальной/рабочей таблице), затем из **корня репозитория** выполните:

```bash
python scripts/build_tn_ved_tree.py
```

Если коды разбиты по **нескольким листам** одной книги:

```bash
python scripts/build_tn_ved_tree.py --sheet all
```

Будет обновлён `frontend/src/catalog/tnVedChildren.generated.ts` (в конце файла — `TN_VED_CHILDREN_BUILD_INFO` с числом строк для контроля полноты).

Из каталога `frontend` можно так: `python ../scripts/build_tn_ved_tree.py` или `npm run build:tn-ved`.

### Ожидаемый формат таблицы

- Первая страница книги (или `--sheet ИмяЛиста` / индекс, либо `--sheet all` для всех листов).
- Две колонки: **код** и **наименование**. Заголовки могут быть на русском или английском, например: `Код`, `код`, `Code`, `ТН ВЭД`, `Наименование`, `название`, `Name`, `Title`.
- В ячейке кода — только цифры (2, 4, 6, 8 или 10 знаков). Пробелы и неразрывные пробелы удаляются автоматически.
- Иерархия восстанавливается так: для каждого кода добавляются недостающие «родительские» уровни (2→4→6→8→10 цифр); родитель для строки — самый длинный код из файла, являющийся префиксом, либо ближайший структурный префикс.

Если колонки не распознаны, берутся **первый столбец** как код и **второй** как наименование.
